## Q&A log

### Q1 — HTML source file structure
- Asked: What do the HTML files contain?
- Captured: These are CNC Job Setup Sheets generated by CAM software (VCarve or similar). The sample file is "260520_gmc_savana_3500_155wb_ew_cargo_Order_1195_Summary_Sheet 9.html" — a setup sheet for a GMC Savana cargo van job. Structure:
  - Header: job name + company logo
  - Sections are `.boxborder` divs, each with a `.boxtitle` heading
  - Sections seen: "Job Layout Sheet 9" (SVG diagram), "Material Setup" (block dimensions, home position, datum), "Toolpaths Summary" (table of toolpath rows)
  - **Toolpath rows** (the main trackable items) live inside `.childindent .fullwidth.FL.nopadding` divs, each with three `.box33` columns: toolpath name | tool name | time estimate
  - Material Setup contains reference info (X/Y/Z dimensions, datum config) — not checklist items
  - The layout SVG is a visual diagram — display only

### Q2 — Completion tracking scope
- Captured: Main trackable items = individual **toolpath rows** in Toolpaths Summary. Each toolpath needs to be marked complete with a date/time stamp. Modal on completion: date/time (editable, pre-filled with now) + operator name field. Material Setup is reference info shown as read-only.

## Summary / key decisions
- Parser targets `.boxborder` elements as sections; within Toolpaths Summary, `.childindent .fullwidth` rows are the items
- Each item shows: toolpath name, tool name, time estimate, checkbox, completion date badge
- Modal fields: date/time override + operator name
- LocalStorage for persistence, keyed by filename + item index
